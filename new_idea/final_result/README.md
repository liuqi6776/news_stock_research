## 🎯 研究概览

本研究在 doubao_result 原策略基础上，新增筹码delta和价格delta特征，对比三种选股策略的表现。

### 核心指标对比

| 策略 | 总收益 | 年化收益 | Sharpe | MDD | 交易次数 | 胜率 |
|------|--------|----------|--------|-----|----------|------|

| **NewIdea S2** | +406.64% | +180.10% | 1.39 | -81.13% | 1030 | 52.91% |

> 回测区间：2023-01 ~ 2026-03 | 初始资金：¥100,000 | 交易成本：0.15%/笔
> 交易规则：T日选股 → T+1开盘买入 → T+2收盘卖出 | 无止盈

## 📋 三种策略详解

### 策略1：doubao_result（基准策略）

**选股逻辑**：
- 模型预测概率 prob > 0.8 → 按prob降序取Top 3
- 若无prob > 0.8的股票 → 取prob最高的Top 1

**模型特征**（5个）：
| 特征 | 说明 |
|------|------|
| `hot_rank_pct` | 同花顺热度分位数（0-1） |
| `chip_concentration` | 筹码集中度 = (cost_85 - cost_15) / (cost_50 + ε) |
| `winner_rate` | 获利盘比例（0-100） |
| `news_market_impact` | 大盘新闻影响分 |
| `news_stock_impact` | 个股新闻影响分 |

**模型**：`doubao_t1t2_model.joblib`（XGBoost分类器）
**标签**：(T+2 close / T+1 open - 1) > 0.04
**股票池**：circ_mv ≤ 100万，排除688开头（科创板）

**核心优势**：
- 严格筛选（prob>0.8），只选高置信度股票
- 平均每天仅1只股票，集中火力
- prob>0.8的7笔交易胜率100%，平均收益12.78%
- prob>0.7的14笔交易胜率92.86%，平均收益10.57%

**劣势**：
- 交易频率低（356笔/397天）
- prob<=0.8的交易占98%，这些交易胜率仅54%

---

### 策略2：NewIdea S2（分位数策略）

**选股逻辑**：
- 无概率阈值，直接取prob最高的Top 3

**模型特征**：同doubao_result（5个基础特征）
**模型**：同doubao_result（共用 `doubao_t1t2_model.joblib`）

**与doubao_result的区别**：
- doubao_result：prob>0.8才选，否则选Top1
- S2：始终选Top3，不论概率高低

**表现分析**：
- S2独有的607笔交易（doubao_result未选的），平均收益仅0.12%，胜率49.92%
- 这些低质量交易严重拖累了整体表现
- prob在[0.3-0.5)区间的775笔交易，平均收益仅0.68%

---

### 策略3：NewIdea S3（特征强化策略）

**选股逻辑**：
- 综合评分 score = prob × 0.7 + winner_rate_norm × 0.2 + (1 - chip_conc_norm) × 0.1
- 按 score 降序取 Top 3

**评分公式详解**：
```
winner_rate_norm = winner_rate / 100.0        # 归一化到0-1
chip_conc_norm = chip_concentration.clip(0,5) / 5.0  # 归一化到0-1
score = prob × 0.7 + winner_rate_norm × 0.2 + (1 - chip_conc_norm) × 0.1
```

**设计意图**：
- prob（70%权重）：模型预测概率为主
- winner_rate（20%权重）：获利盘高→抛压小
- (1-chip_concentration)（10%权重）：筹码越分散→越不容易被砸

**表现分析**：
- S3选出的股票prob均值0.4265，低于S2的0.4511
- score排序引入了非prob维度的噪音，反而降低了选股质量
- 筹码集中度和获利盘与短期收益的相关性弱于prob

## 🚀 提升方向

### 方向1：优化选股参数（最直接）

**推荐组合**：prob ≥ 0.4 + Top 1 + No TP
- 预期收益：+2146%
- 预期Sharpe：4.81
- 实现方式：修改 `strategy_code.py` 中的 `select()` 函数

### 方向2：动态止盈

根据概率设置不同止盈点：
- prob ≥ 0.7 → 止盈20%或No TP（高置信度，让利润奔跑）
- prob 0.5-0.7 → 止盈15%
- prob < 0.5 → 止盈12%

### 方向3：仓位管理

按概率加权分配仓位：
```python
weight = prob / sum(probs)  # 概率越高仓位越重
```

### 方向4：风控优化

- 单日亏损超5% → 次日仓位减半
- 连续3天亏损 → 暂停1天
- 最大回撤超20% → 仓位降至30%

### 方向5：特征工程

当前delta特征未带来显著提升，建议：
- 移除delta特征，回归5特征模型
- 或尝试其他特征：换手率、量比、板块动量等
- S3的score公式需要重新调参或引入更有效的辅助因子

## 📋 完整交易规则

### 时间线（三天一个周期）

```
┌─────────────────────────────────────────────────────────┐

                            ↓

│  ↓                                                        │
│  1. 检查：是否开盘涨停？                                   │

│     ├─ 是 → T+3日开盘卖出                                 │

│  2. 无主动止盈（当前最优配置）                             │
└─────────────────────────────────────────────────────────┘
```

### 买入规则（T+1日早盘）

**买入时间**：T+1日开盘（9:30）
**买入价格**：开盘价
**涨停检查**：
- 主板/中小板：涨停 = 前收盘价 × 1.1
- 创业板/科创板：涨停 = 前收盘价 × 1.2
- 开盘价 ≥ 涨停价 → 放弃买入

**仓位分配**：等权分配（选N只，每只1/N仓位）

### 卖出规则（T+2日）

**跌停处理**：
- 检查T+2日最高价是否等于跌停价
- 若跌停 → T+3日开盘卖出
- 否则 → T+2日收盘价卖出

**交易成本**：每笔0.15%

### 股票池过滤

| 过滤条件 | 说明 |
|----------|------|
| circ_mv ≤ 100万 | 流通市值不超过100万（万元单位） |
| 排除688开头 | 排除科创板股票 |
| 涨停不买 | T+1日开盘涨停则放弃 |

## 🕐 时间序列特征增强策略（2026-04-19 更新）

### 研究背景

前两轮优化（Enhanced多方案优化、Delta特征研究）发现：
- 新增静态特征（换手率/PE/PB等）严重拖累表现
- Delta特征单独使用效果不佳
- **核心问题**：现有特征都是"当天数字"，没有学习到"变化趋势"

本轮引入**时间序列（TS）特征**，捕捉价格和筹码的动态变化规律，在doubao_result的356笔交易基础上进行TS过滤和重排序。

### TS特征设计（严格无数据泄露）

所有特征均使用 **T-1 及更早数据** 计算，预测目标为 T+2 收益，确保无信息泄露：

| 特征 | 计算方式 | 金融含义 |
|------|----------|----------|
| `ret_1d` | T日close / T-1日close - 1 | 1日涨幅（追高信号） |
| `delta_winner_rate` | T日winner_rate - T-1日winner_rate | 获利盘变化（筹码改善/恶化） |
| `delta_chip_conc` | T日chip_concentration - T-1日chip_concentration | 筹码集中度变化 |
| `ret_5d` | T日close / T-5日close - 1 | 5日累计涨幅 |
| `ma5_dist` | T日close / MA5(T-1~-5) - 1 | 偏离5日均线程度 |
| `ts_score_v2` | 加权组合评分 | 综合TS信号 |

**ts_score_v2 公式**：
```
ts_score_v2 = -|ret_1d| × 0.5 + delta_winner_rate × 3.0
              -|delta_chip_conc| × 1.5 - |ret_5d| × 0.2
              -|ma5_dist| × 0.3
```

设计逻辑：
- **ret_1d 负权重**：昨日涨幅越大，今日追高风险越高
- **delta_winner_rate 正权重**：获利盘上升 = 筹码结构改善
- **delta_chip_conc 负权重**：筹码集中度变化大 = 不稳定
- **ret_5d / ma5_dist 负权重**：偏离均线越远，回归风险越大

### TS特征效果深度分析

#### 1. delta_winner_rate > 0 的效果

**解读**：**这是最有效的风控过滤器**。昨日涨幅>5%的股票，今日追高买入后T+2被套概率大。过滤后最大回撤从-68.6%降至-29.7%，风险大幅降低。

### 关键发现总结

1. **TS特征显著提升风险调整后收益**：Sharpe从5.49最高提升至67.18（TS_score>0方案）

2. **不追高是最有效的风控**：ret_1d < 0.05将MDD从-68.6%降至-29.7%，回撤减少57%

3. **筹码改善是强Alpha信号**：delta_winner_rate > 0使Sharpe翻倍（5.49→11.95）

4. **综合TS评分效果最强**：ts_score_v2 > 0在保留2000%+收益的同时，Sharpe提升12倍

5. **交易数量与质量的权衡**：TS过滤减少交易数量，但每笔交易质量显著提升

6. **止盈25%是TS策略的最优止盈点**：配合TS过滤，TP25%进一步提升Sharpe

### ⚠️ 注意事项

- **小样本偏差**：交易数<50的方案（如WR>-2 & R1d<0.03，32笔）Sharpe异常高，统计意义有限
- **数据期限**：TS_score>0方案仅覆盖0.62年（156个交易日），需更长周期验证
- **推荐优先级**：TS_score>0（156笔）> R1d<0.05（144笔）> WR>0（175笔），交易数越多越稳健
- **实盘建议**：先用R1d<0.05+TP25%作为稳健方案，同时观察TS_score>0的实盘表现
